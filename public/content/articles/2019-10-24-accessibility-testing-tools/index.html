Accessibility testing tools

I recently came across a question in the Design Systems Slack community from Sonali Zaveri relating to accessibility testing tools.

What tools does everyone use to test accessibility? I’m hoping to put together a testing playbook for my team. I want to detail out what tools to use and how, and what manual testing to perform (ie. Run Chrome Lighthouse, Check tab order, Increase your font size, etc.).

It's a great question. I certainly have my go to list of tools bookmarked. It was interesting to see the number of different tools that were suggested. I thought it would be helpful to document the various tools for future reference.

Developer tools

  • Google Lighthouse – Lighthouse is an open-source, automated tool for improving the quality of web pages. You can run it against any web page, public or requiring authentication. It has audits for performance, accessibility, progressive web apps, and more.
  • Landmarks – Allows you to navigate a web page via WAI-ARIA landmarks, using the keyboard or a pop-up menu.

Browser extensions and plugins

  • Chrome Lens – Visual impairment simulation and auditing tools to aid accessibility development.
  • aXe - Web accessibility testing – Accessibility checker for WCAG 2 and Section 508 accessibility.
  • aXe Coconut – Experimental version of the popular aXe - Web accessibility checker.
  • WAVE – WAVE is a web accessibility evaluation tool developed by WebAIM.org. It provides visual feedback about the accessibility of your web content by injecting icons and indicators into your page.
  • Alix HTML linter – Alix allows you to lint your HTML for accessibility issues simply by applying a stylesheet that makes use of advanced CSS selectors.
  • NoCoffee vision simulator – NoCoffee can be helpful for understanding the problems faced by people with slight to extreme vision problems
  • Colour contrast analyser – Analyze a web page or portion of a web page for conformance with WCAG 2.0 Color Contrast requirements.
  • Funkify – Funkify helps you experience the web and interfaces through the eyes of extreme users with different abilities and disabilities.

Bookmarklets

  • Akbar – Simulates various vision scenarios.
  • Nerde Focus – Quickly debug focus problems when testing a page for accessibility issues.
  • tota11y – tota11y helps visualise how your site performs with assistive technologies.
  • aXe HTML Report – A bookmarklet for generating and downloading aXe test results in HTML format
  • Doc Outline Audit – This small script analyses the heading outline of the document (or given container), and report possible errors.
  • ANDI – ANDI (Accessible Name & Description Inspector) is a free tool to test web content for accessibility.
  • Form autocomplete tool – Bookmarklet to fill out forms when testing by simply clicking on it.
  • Caret Browsing – Browse the text of web pages using arrow keys.

Screen readers

  • NVDA
  • NVDA quick key reference
  • Voice Over (User Guide)
  • Voice Over quick key reference
  • JAWS

Applications

  • koa11y – Koa11y is a desktop app that allows you to automatically detect accessibility (a11y) issues on webpages. Available on Windows, Mac OS and Linux.
  • Contrast Color accessibility – A Mac OS menu bar app that helps check for WCAG contrast ratios.
  • Color Oracle – Color Oracle is a free color blindness simulator for Windows, Mac and Linux.
  • Sort Site – SortSite is a one-click web site testing tool that checks each page against more than 1200 standards based checkpoints.
